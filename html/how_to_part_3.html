Premiers pas sur Bitbucket

Bitbucket est un outil de gestion des dépôts de code source Git et Mercurial, sur serveur ou cloud.

C'est ici que vous allez pouvoir récupérer et mettre à jour vos projets. Mais accéder aux dépots, vous devez tout d'abord générer une paire de clés SSH et partager la publique avec bitbucket. Pour plus d'informations sur les clés ssh et leur fonctionnemnt cliquez ici Rendez vous sur l'onglet en bas à gauche et cliquez sur "View profile"

Après cela, cliquez sur l'onglet "Settings" puis sur "SSH keys"

Générer votre clé SSH.
Lors de la génération de votre clé dans votre terminal (j'espère que vous avez télécharger terminator !), on vous demandera l'emplacement de votre clé. Vous laisserez celle par défaut en appuyant sur "Entrée". On vous demandera également un mot de passe, tapez "Entrée" pour ne pas en mettre ! puis tapez une deuxième fois sur "Entrée" pour confirmer.

Ajouter ensuite votre clé sur bitbucket, en cliquant sur le bouton "Add key"

Puis copiez-collez votre clé publique

Remplissez le champ "label" avec la première lettre de votre prenom et votre nom. Remarque au passage : Si vous avez plusieurs distributions, ajoutez à votre label un "_" suivi du nom de la ditribution. Exemple : mdjabri_ubuntu.

Projet Empty_App ou Existant?

Maintenant que votre environnement est prêt. Nous allons pouvoir commmencer les choses sérieuses. Selon le projet qui vous sera affecté, c'est-à-dire l'application que vous devrez réaliser pour le client, vous serez toujours ammenés à partir d'une base existante. Reste à définir quel type de projet vous conviendra le mieux. En règle général, on vous le dira explicitement. Mais dans le cas contraire sachez que vous avez deux possibilités:

  • Vous développez une nouvelle application qui peut comporter ou non un nouveau sous module et dans ce cas vous partirez du projet nommé "empty_app" dans bitbucket
  • Vous développez une application qui ressemble à une application existante. Dans ce cas là vous allez "forker" l'application existante

Projet Empty App

Aucune application déjà réalisée ne comporte de sous modules dont vous avez besoin ou ne ressemble à celle que vous devez faire, alors vous devez partir d'un projet vide! empty_app est constitué d'éléments de base pour effectué une application. Voyons ensemble de quoi elle est constituée.

Empty app est une application dont le framework est constitué d'éléments de base. Elle comporte:

  • Un dossier "dialog" où sont rangés les topics (C'est dans un fichier .top que vous écrirez les règles pour doter votre robot d'une habilité à dialoguer en vous aidant des concepts déjà écrits )
  • Un dossier templating/data qui comporte deux fichiers .json nécessaires à l'utilisation de vos "routines"
  • Un dossier html qui sera utilisé pour l'affichage de la tablette
  • Un dossier libs où vous trouverez le main.py
  • Un dossier module_core qui est composé de fonctionnalités importantes du robot (Scheduler.py, poeple_perception.py, services_handler.py, services_instanciation.py, services_launcher.py etc...)
  • Un dossier module_menu
  • Un fichier manifest.xml. Vous en avez besoin pour rendre votre Application sous forme de .pkg

Projet existant

Des applications, il y en a pleins! Si vous devez repartir d'un projet existant, trouvez celui qui sera le plus adapté. Ces applications sont composés des fonctionnalités de base que l'on retrouve dans empty_app mais avec des modules supplémentaires. On pourra donc trouver:

  • Lilly_app qui est composée d'un menu général qui comporte un module de satisfaction, (enquête de satisfaction), un module posture (module pour faire un selfie avec Pepper), un module show_video (integration de vidéos)
  • Socoda, Niort_numeric_selfie_app ou innovhealth_selfie_app qui est sont des applications dont l'unique but est de faire des selfies. (Souvent utilisés pour des conventions/events donc pas besoin de menu principal).
  • Klepierre...
  • Elior_lab est une application complète qui comporte deux enquêtes de satisfaction, un module selfie, un module présentation (présentation de l'entreprise du client), un module plan (affichage d'un plan + dialogue), un module video
  • InnovHealth

Voilà pour ce qui est des applications utilisable pour réaliser vos projets.

Un peu de pratique

C'est bien beau tout ce discours Mohamed, mais comment on s'y prends concrètement ?! Eh bien mes chers robotopiens ne soyez pas si préssé car c'est là qu'arrive le plus dur croyez moi. Premièrement vous avez besoin de "forker" le projet. Pour cela rien de très compliquer. Prenons l'exemple d'Elior_lab... Allez sur la barre de recherche sur bitbucket et tapez "elior_lab" puis cliquez dessus. Puis dans la barre de recherche de votre navigateur, effacer le contenu écrit avant elior_lab/ et rajoutez le mot fork comme suit:

Vous serez reconduit sur la page suivante, où vous devrez changer l'Owner et choisir "consertoroboticteam". Dans l'onglet Project, choisissez "Applications" et modifier le nom de l'application. Après cela, il ne vous reste plus qu'à cliquer sur l'onglet "Fork repository" et votre projet sera crée.

Il ne vous reste plus qu'à aller rechercher votre projet et "git cloner" votre repository dans votre répertoire de travail.
